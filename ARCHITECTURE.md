# cuda_sim - Architecture

## Goal

Run CUDA kernels on CPU without a GPU. User provides `.cu` source code, our toolchain compiles it into a native CPU executable.

**Use cases**: debugging kernels without GPU, running CUDA tests in CI, teaching CUDA programming model.

**Prerequisite**: CUDA Toolkit (nvcc) must be installed. GPU is NOT required.

---

## Approach: PTX-level Translation

We translate at the **PTX level** (CUDA's intermediate representation), not at the source level.

Why PTX, not source-level?
- Source-level simulation requires reimplementing hundreds of CUDA SDK functions
- nvcc compiles all SDK functions down to ~200 PTX instructions
- Translating PTX is mechanical and complete — no coverage gaps

```
CUDA's normal flow:
  .cu → nvcc → PTX → SASS → GPU execution

Our flow:
  .cu → nvcc -ptx → PTX → ptx2cpp.py → C++ → g++ → CPU execution
                          ↑
                    our translator
```

---

## Compilation Pipeline

```
Step 1             Step 2                Step 3             Step 4
  .cu      →    kernel.ptx     →    kernel_cpu.cpp   →   executable
         nvcc -ptx          ptx2cpp.py             g++

(NVIDIA)        (text file,       (generated C++,      (native CPU
                 readable)         one line per          binary)
                                   PTX instruction)
```

### Typical CUDA project structure

```
my_project/
├── main.cpp        ← host code (CPU): cudaMalloc, cudaMemcpy, launch kernel
├── kernel.cu       ← device code (GPU): __global__ kernel functions
└── Makefile
```

### Cross-platform: Docker for PTX generation

Since nvcc is not available on macOS (NVIDIA dropped support in 2020), we use Docker to run nvcc. Only the PTX generation step needs Docker — the rest runs natively.

```
Any platform (Mac / Linux / Windows)
├── Docker (nvidia/cuda image) → nvcc -ptx → kernel.ptx   ← only this needs Docker
├── ptx2cpp.py                 → kernel_cpu.cpp            ← pure Python, native
└── g++ / clang++              → executable                ← native compiler
```

Quick start:
```bash
# One-command build (Docker + translate + compile)
./cuda-sim-build.sh kernel.cu main.cpp -o app

# Or step by step:
./docker/generate_ptx.sh kernel.cu              # Docker: .cu → .ptx
python3 tools/ptx2cpp.py kernel.ptx -o kernel_cpu.cpp  # translate
g++ -std=c++17 -Iinclude/ main.cpp kernel_cpu.cpp -o app  # compile
./app                                            # run on CPU
```

> Note: The Docker image (`nvidia/cuda:12.6.0-devel`) is ~3.5GB on first pull, but doesn't require a GPU — it only uses nvcc as a compiler.

### Normal CUDA compilation (requires GPU)

```bash
# nvcc handles both: .cpp goes to gcc, .cu goes through CUDA pipeline
nvcc main.cpp kernel.cu -lcudart -o app
./app    # runs kernel on GPU
```

Internally nvcc does: `kernel.cu → PTX → SASS → fatbinary`, embeds GPU binary in the executable, compiles `main.cpp` with gcc, links everything with `libcudart.so` which talks to GPU driver at runtime.

### Our approach (no GPU needed)

```bash
# Step 1: Generate PTX from kernel source (requires nvcc, not GPU)
nvcc -ptx kernel.cu -o kernel.ptx

# Step 2: Translate PTX to C++ (our tool)
python ptx2cpp.py kernel.ptx -o kernel_cpu.cpp

# Step 3: Compile host code + translated kernel + our runtime (pure CPU)
g++ -O2 main.cpp kernel_cpu.cpp -I include/ -lcudart_sim -o app

# Step 4: Run on CPU
./app
```

We only run nvcc on `kernel.cu` to get PTX. The host code `main.cpp` is compiled directly by g++ — it calls our `libcudart_sim` instead of the real `libcudart.so`.

## How PTX Translation Works

### PTX format

Every PTX instruction follows a regular pattern:

```
opcode.type  dest, src1, src2;
```

Example PTX (generated by `nvcc -ptx` from a vectorAdd kernel):

```
.visible .entry vectorAdd(
    .param .u64 a,
    .param .u64 b,
    .param .u64 c,
    .param .u32 n
) {
    .reg .f32   %f<4>;
    .reg .u32   %r<8>;
    .reg .u64   %rd<8>;
    .reg .pred  %p<2>;

    mov.u32       %r1, %tid.x;
    mov.u32       %r2, %ctaid.x;
    mov.u32       %r3, %ntid.x;
    mad.lo.s32    %r4, %r2, %r3, %r1;
    ld.param.u32  %r5, [n];
    setp.ge.s32   %p1, %r4, %r5;
    @%p1 bra      END;

    ld.param.u64  %rd1, [a];
    mul.wide.u32  %rd3, %r4, 4;
    add.u64       %rd4, %rd1, %rd3;
    ld.global.f32 %f1, [%rd4];

    ld.param.u64  %rd5, [b];
    add.u64       %rd6, %rd5, %rd3;
    ld.global.f32 %f2, [%rd6];

    add.f32       %f3, %f1, %f2;

    ld.param.u64  %rd7, [c];
    add.u64       %rd8, %rd7, %rd3;
    st.global.f32 [%rd8], %f3;
END:
    ret;
}
```

### Translation table

Each PTX instruction maps to one C++ statement:

```
PTX                                 C++
──────────────────────────          ───────────────────────────────────
mov.u32     %r1, %tid.x            r[1] = tid_x;
mov.u32     %r2, %ctaid.x          r[2] = ctaid_x;
mov.u32     %r3, %ntid.x           r[3] = ntid_x;
mad.lo.s32  %r4, %r2, %r3, %r1    r[4] = (int32_t)r[2] * (int32_t)r[3] + (int32_t)r[1];
ld.param.u32 %r5, [n]              r[5] = param_n;
setp.ge.s32  %p1, %r4, %r5         p[1] = ((int32_t)r[4] >= (int32_t)r[5]);
@%p1 bra     END                   if (p[1]) goto END;

ld.param.u64  %rd1, [a]            rd[1] = param_a;
mul.wide.u32  %rd3, %r4, 4         rd[3] = (uint64_t)r[4] * 4;
add.u64       %rd4, %rd1, %rd3     rd[4] = rd[1] + rd[3];
ld.global.f32 %f1, [%rd4]          f[1] = *(float*)(rd[4]);

add.f32       %f3, %f1, %f2        f[3] = f[1] + f[2];
st.global.f32 [%rd8], %f3          *(float*)(rd[8]) = f[3];

cvt.f32.s32  %f1, %r1              f[1] = (float)(int32_t)r[1];
sin.approx.f32 %f2, %f1            f[2] = sinf(f[1]);
atom.global.add.s32 %r1,[%rd2],%r3 r[1] = atomic_add((int*)rd[2], r[3]);
```

~200 PTX instructions total, but most are the same pattern with different types (f32/f64/s32/u32/u64). Actual translation rules: ~50.

### Generated C++

ptx2cpp.py produces:

```cpp
// kernel_cpu.cpp (auto-generated from kernel.ptx)
#include <cstdint>
#include <cmath>
#include <atomic>
#include "cuda_sim/runtime.h"

void vectorAdd_kernel(uint64_t param_a, uint64_t param_b,
                      uint64_t param_c, uint32_t param_n,
                      uint32_t tid_x, uint32_t tid_y, uint32_t tid_z,
                      uint32_t ctaid_x, uint32_t ctaid_y, uint32_t ctaid_z,
                      uint32_t ntid_x, uint32_t ntid_y, uint32_t ntid_z) {
    float    f[4]  = {};
    uint32_t r[8]  = {};
    uint64_t rd[8] = {};
    bool     p[2]  = {};

    r[1] = tid_x;                                                // mov.u32 %r1, %tid.x
    r[2] = ctaid_x;                                              // mov.u32 %r2, %ctaid.x
    r[3] = ntid_x;                                               // mov.u32 %r3, %ntid.x
    r[4] = (int32_t)r[2] * (int32_t)r[3] + (int32_t)r[1];      // mad.lo.s32
    r[5] = param_n;                                              // ld.param.u32
    p[1] = ((int32_t)r[4] >= (int32_t)r[5]);                    // setp.ge.s32
    if (p[1]) goto END;                                          // @%p1 bra END

    rd[1] = param_a;                                             // ld.param.u64
    rd[3] = (uint64_t)r[4] * 4;                                 // mul.wide.u32
    rd[4] = rd[1] + rd[3];                                      // add.u64
    f[1]  = *(float*)(rd[4]);                                    // ld.global.f32

    rd[5] = param_b;                                             // ld.param.u64
    rd[6] = rd[5] + rd[3];                                      // add.u64
    f[2]  = *(float*)(rd[6]);                                    // ld.global.f32

    f[3]  = f[1] + f[2];                                        // add.f32

    rd[7] = param_c;                                             // ld.param.u64
    rd[8] = rd[7] + rd[3];                                      // add.u64  (NOTE: rd[8] out of bounds, real PTX uses larger arrays)
    *(float*)(rd[8]) = f[3];                                     // st.global.f32
END:
    return;
}
```

## Thread Simulation

The generated kernel function handles a single thread. Our runtime wraps it in a grid/block loop:

```cpp
// Generated by ptx2cpp.py alongside the kernel
void vectorAdd_launch(uint64_t a, uint64_t b, uint64_t c, uint32_t n,
                      dim3 grid, dim3 block) {
    for (uint32_t bz = 0; bz < grid.z; ++bz)
    for (uint32_t by = 0; by < grid.y; ++by)
    for (uint32_t bx = 0; bx < grid.x; ++bx)
        for (uint32_t tz = 0; tz < block.z; ++tz)
        for (uint32_t ty = 0; ty < block.y; ++ty)
        for (uint32_t tx = 0; tx < block.x; ++tx)
            vectorAdd_kernel(a, b, c, n,
                             tx, ty, tz,       // threadIdx
                             bx, by, bz,       // blockIdx
                             block.x, block.y, block.z);  // blockDim
}
```

`dim3` supports 1D, 2D, or 3D. Unused dimensions default to 1:
- 1D: `dim3(256)` → x=256, y=1, z=1
- 2D: `dim3(16, 16)` → x=16, y=16, z=1
- 3D: `dim3(8, 8, 8)` → x=8, y=8, z=8

## Host Runtime (libcudart_sim)

A lightweight library that provides CUDA Runtime API functions:

```
CUDA API              Our implementation
──────────            ──────────────────
cudaMalloc()      →   malloc + allocation tracking (leak detection)
cudaFree()        →   free + validate pointer
cudaMemcpy()      →   memcpy
cudaMemset()      →   memset
cudaDeviceSync()  →   no-op (already synchronous)
cudaGetLastError()→   thread-local error code
```

The host code (`main.cpp`) calls these APIs. Instead of launching a kernel on GPU, it calls the generated `_launch()` wrapper function.

## Hello World: Complete Example

### 1. User writes CUDA kernel (`kernel.cu`)

```cuda
extern "C"
__global__ void vectorAdd(const float* a, const float* b, float* c, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i < n) {
        c[i] = a[i] + b[i];
    }
}
```

> `extern "C"` prevents C++ name mangling, making PTX function names predictable.

### 2. Generate PTX

```bash
$ nvcc -ptx kernel.cu -o kernel.ptx
```

Produces something like:

```
.visible .entry vectorAdd(
    .param .u64 vectorAdd_param_0,
    .param .u64 vectorAdd_param_1,
    .param .u64 vectorAdd_param_2,
    .param .u32 vectorAdd_param_3
) {
    .reg .pred  %p<2>;
    .reg .f32   %f<4>;
    .reg .b32   %r<6>;
    .reg .b64   %rd<11>;

    ld.param.u64  %rd1, [vectorAdd_param_0];
    ld.param.u64  %rd2, [vectorAdd_param_1];
    ld.param.u64  %rd3, [vectorAdd_param_2];
    ld.param.u32  %r2,  [vectorAdd_param_3];
    mov.u32       %r3,  %ctaid.x;
    mov.u32       %r4,  %ntid.x;
    mov.u32       %r5,  %tid.x;
    mad.lo.s32    %r1,  %r3, %r4, %r5;
    setp.ge.s32   %p1,  %r1, %r2;
    @%p1 bra      $L__BB0_2;

    cvta.to.global.u64 %rd4, %rd1;
    mul.wide.s32  %rd5, %r1, 4;
    add.s64       %rd6, %rd4, %rd5;
    ld.global.f32 %f1,  [%rd6];
    cvta.to.global.u64 %rd7, %rd2;
    add.s64       %rd8, %rd7, %rd5;
    ld.global.f32 %f2,  [%rd8];
    add.f32       %f3,  %f1, %f2;
    cvta.to.global.u64 %rd9, %rd3;
    add.s64       %rd10, %rd9, %rd5;
    st.global.f32 [%rd10], %f3;

$L__BB0_2:
    ret;
}
```

### 3. Translate PTX to C++

```bash
$ python tools/ptx2cpp.py kernel.ptx -o kernel_cpu.cpp
```

Generates:

```cpp
// kernel_cpu.cpp — auto-generated by ptx2cpp.py
#include <cstdint>
#include "cuda_sim/runtime.h"

static void vectorAdd_thread(
    uint64_t param_0, uint64_t param_1, uint64_t param_2, uint32_t param_3,
    uint32_t tid_x, uint32_t tid_y, uint32_t tid_z,
    uint32_t ctaid_x, uint32_t ctaid_y, uint32_t ctaid_z,
    uint32_t ntid_x, uint32_t ntid_y, uint32_t ntid_z)
{
    bool     p[2]  = {};
    float    f[4]  = {};
    uint32_t r[6]  = {};
    uint64_t rd[11] = {};

    rd[1] = param_0;                                         // ld.param.u64
    rd[2] = param_1;                                         // ld.param.u64
    rd[3] = param_2;                                         // ld.param.u64
    r[2]  = param_3;                                         // ld.param.u32
    r[3]  = ctaid_x;                                         // mov.u32 %ctaid.x
    r[4]  = ntid_x;                                          // mov.u32 %ntid.x
    r[5]  = tid_x;                                           // mov.u32 %tid.x
    r[1]  = (int32_t)r[3] * (int32_t)r[4] + (int32_t)r[5]; // mad.lo.s32
    p[1]  = ((int32_t)r[1] >= (int32_t)r[2]);                // setp.ge.s32
    if (p[1]) goto L__BB0_2;                                 // @%p1 bra

    rd[4] = rd[1];                                           // cvta.to.global.u64
    rd[5] = (int64_t)(int32_t)r[1] * 4;                     // mul.wide.s32
    rd[6] = rd[4] + rd[5];                                   // add.s64
    f[1]  = *(float*)(rd[6]);                                // ld.global.f32
    rd[7] = rd[2];                                           // cvta.to.global.u64
    rd[8] = rd[7] + rd[5];                                   // add.s64
    f[2]  = *(float*)(rd[8]);                                // ld.global.f32
    f[3]  = f[1] + f[2];                                     // add.f32
    rd[9] = rd[3];                                           // cvta.to.global.u64
    rd[10] = rd[9] + rd[5];                                  // add.s64
    *(float*)(rd[10]) = f[3];                                // st.global.f32

L__BB0_2:
    return;
}

// Thread simulation wrapper
extern "C"
void vectorAdd_launch(uint64_t p0, uint64_t p1, uint64_t p2, uint32_t p3,
                      cuda_sim::dim3 grid, cuda_sim::dim3 block) {
    for (uint32_t bz = 0; bz < grid.z; ++bz)
    for (uint32_t by = 0; by < grid.y; ++by)
    for (uint32_t bx = 0; bx < grid.x; ++bx)
        for (uint32_t tz = 0; tz < block.z; ++tz)
        for (uint32_t ty = 0; ty < block.y; ++ty)
        for (uint32_t tx = 0; tx < block.x; ++tx)
            vectorAdd_thread(p0, p1, p2, p3,
                             tx, ty, tz,
                             bx, by, bz,
                             block.x, block.y, block.z);
}
```

### 4. User writes host code (`main.cpp`)

```cpp
#include <cstdio>
#include <cstdint>
#include "cuda_sim/cuda_runtime_api.h"
#include "cuda_sim/runtime.h"

// Declaration of the generated launch function
extern "C"
void vectorAdd_launch(uint64_t, uint64_t, uint64_t, uint32_t,
                      cuda_sim::dim3, cuda_sim::dim3);

int main() {
    const int N = 1024;
    float h_a[N], h_b[N], h_c[N];
    for (int i = 0; i < N; i++) {
        h_a[i] = (float)i;
        h_b[i] = (float)(i * 2);
    }

    // Allocate "device" memory (actually CPU memory)
    float *d_a, *d_b, *d_c;
    cudaMalloc((void**)&d_a, N * sizeof(float));
    cudaMalloc((void**)&d_b, N * sizeof(float));
    cudaMalloc((void**)&d_c, N * sizeof(float));

    // Copy host → "device"
    cudaMemcpy(d_a, h_a, N * sizeof(float), cudaMemcpyHostToDevice);
    cudaMemcpy(d_b, h_b, N * sizeof(float), cudaMemcpyHostToDevice);

    // Launch kernel on CPU
    int blockSize = 256;
    int numBlocks = (N + blockSize - 1) / blockSize;
    vectorAdd_launch((uint64_t)d_a, (uint64_t)d_b, (uint64_t)d_c, N,
                     cuda_sim::dim3(numBlocks), cuda_sim::dim3(blockSize));

    // Copy "device" → host
    cudaMemcpy(h_c, d_c, N * sizeof(float), cudaMemcpyDeviceToHost);

    // Verify
    int errors = 0;
    for (int i = 0; i < N; i++) {
        float expected = (float)i + (float)(i * 2);
        if (h_c[i] != expected) {
            printf("FAIL at %d: got %f, expected %f\n", i, h_c[i], expected);
            errors++;
        }
    }
    if (errors == 0) {
        printf("PASS: all %d elements correct\n", N);
    }

    cudaFree(d_a);
    cudaFree(d_b);
    cudaFree(d_c);
    return errors > 0 ? 1 : 0;
}
```

### 5. Build and run

```bash
$ g++ -O2 -std=c++17 -Iinclude/ main.cpp kernel_cpu.cpp runtime/memory.cpp -o app
$ ./app
PASS: all 1024 elements correct
```

## CUDA SDK Compatibility

CUDA is only needed at **compile time** to generate PTX. The final product is a **pure CPU program** with zero CUDA dependency at runtime.

```
                     Compile time                          Runtime
                     (needs nvcc)                          (no CUDA at all)
                     ─────────────                         ────────────────
.cu → nvcc -ptx → kernel.ptx → ptx2cpp.py → kernel_cpu.cpp ─┐
                                                              ├→ g++ → pure CPU binary
                                              main.cpp ───────┘
                                              libcudart_sim (our malloc/memcpy wrappers)
```

| | Compile time | Runtime |
|---|---|---|
| CUDA Toolkit (nvcc) | Required | NOT needed |
| GPU Driver | NOT needed | NOT needed |
| GPU Hardware | NOT needed | NOT needed |
| libcudart.so | NOT needed | NOT needed (we provide our own) |

The only role of CUDA in our pipeline is: **nvcc as a `.cu` → `.ptx` compiler**. After that, CUDA is completely gone.

## Shared Memory & Synchronization

When a kernel uses shared memory (`.shared` declarations in PTX), the execution model changes:

```
Without shared memory (sequential — fast):
  for each block:
      for each thread:
          call kernel_thread(...)     ← simple function calls

With shared memory (multi-threaded):
  for each block:
      allocate shared_mem buffer
      create barrier(num_threads)
      spawn N real OS threads         ← std::thread
          each calls kernel_thread(..., shared_mem, &barrier)
      join all threads
```

- `bar.sync` translates to `barrier.arrive_and_wait()` — a real thread barrier
- Shared memory is a `uint8_t[]` buffer shared by all threads in a block
- `ld.shared`/`st.shared` translate to pointer dereferences into the shared buffer
- Kernels without shared memory use the sequential model (no threading overhead)

## Scope and Limitations

**Works well for** (verified with real nvcc-generated PTX):
- Vector operations (vectorAdd) — 1D grid, global memory
- Histogram — atomicAdd
- Parallel reduction — shared memory, __syncthreads()
- Matrix multiply — 2D shared memory tiles, 2D grid/block, multiple barriers

**Limited support**:
- Warp-level primitives (shfl, ballot) — requires simulating warp concept
- Dynamic parallelism (kernel launching kernel)

**Not supported**:
- Inline PTX assembly beyond standard instructions
- Texture/surface memory
- CUDA streams (async execution)
- cuBLAS, cuFFT, cuDNN (separate libraries, not kernel-level)
- Pre-compiled binaries (SASS only, no PTX available)
